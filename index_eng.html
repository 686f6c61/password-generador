What is password entropy?

Password entropy is a measure of its strength or unpredictability. In the context of computer security, entropy is expressed in bits and represents the amount of information or randomness contained in a password.

A password with high entropy is more difficult to guess or crack through brute force or dictionary attacks. Entropy is calculated using the following formula:

E = L * log2(R)

Where:

  • E is the entropy in bits
  • L is the length of the password
  • R is the size of the possible character set

For example, for an 8-character password that uses lowercase and uppercase letters (52 characters) and numbers (10 characters), the calculation would be:

In the case of word-based passwords, the calculation is slightly different. The dictionary size is used instead of the character set:

E = N * log2(D)

Where:

  • N is the number of words in the password
  • D is the dictionary size

Higher entropy indicates a more secure password. In general, a minimum entropy of 60 bits is recommended for adequate security, although higher values may be required for highly sensitive information.

What is brute force and its variants?

security Brute force attack

A brute force attack is a method of password cracking that involves systematically trying all possible combinations of characters until the correct one is found. Although it is a guaranteed method for finding any password, it can be extremely slow for long and complex passwords.

content_paste Credential stuffing

Credential stuffing is a technique that uses username and password pairs leaked from a security breach to try and access other accounts. This method exploits users' tendency to reuse the same credentials across multiple sites.

book Dictionary attack

A dictionary attack uses a predefined list of common words and phrases to try to guess the password. It is faster than pure brute force but less exhaustive. It is especially effective against passwords that use common words.

shuffle Hybrid brute force attacks

Hybrid attacks combine elements of dictionary and brute force attacks. They may start with words from a dictionary and then apply common transformations (such as adding numbers at the end) or combine dictionary words with brute force-generated character sequences.

flip_to_back Reverse brute force attacks

Instead of attempting to crack a specific password, reverse brute force attacks generate hashes of common passwords and compare them to a database of stolen hashes. This method is effective at cracking multiple passwords simultaneously if access to a hash database is available.

memory GPU-accelerated brute force

GPU acceleration uses the processing power of graphics cards to perform brute force attacks much faster than traditional CPUs. GPUs are particularly efficient at performing many simple calculations in parallel, making them ideal for quickly testing a large number of password combinations.
